# 01 — Setup

## Installation

nnU-Net v2 (package name `nnunetv2`, current version **v2.7.0**) requires Python >= 3.10.

> **Critical: Install PyTorch FIRST.** The official nnU-Net docs warn: *"DO NOT JUST
> install nnunetv2 WITHOUT PROPERLY INSTALLING PYTORCH FIRST."* Installing nnunetv2
> can silently replace a CUDA-enabled PyTorch build with a CPU-only one. Always install
> PyTorch manually first via https://pytorch.org, selecting the correct CUDA version for
> your hardware.

```bash
# Step 1: Install PyTorch with CUDA (check https://pytorch.org for exact command)
uv add torch torchvision --index-url https://download.pytorch.org/whl/cu121

# Step 2: Install nnU-Net
uv add nnunetv2
```

For development (editable install — required if you need custom trainers in the source tree):
```bash
git clone https://github.com/MIC-DKFZ/nnUNet.git
uv add --editable ./nnUNet
```

### Known Dependency Issues

| Issue | Details |
|---|---|
| **`torch!=2.9.*`** | nnU-Net v2.7.0 explicitly excludes torch 2.9.x due to a severe performance regression with 3D convolutions under AMP. Use torch 2.8.x or lower. |
| **PyTorch 2.6 `weights_only`** | PyTorch 2.6 changed `torch.load()` default to `weights_only=True`, which can break checkpoint loading. If you encounter `UnpicklingError`, ensure you are on the latest nnunetv2 version. |
| **`blosc2>=3.0.0b1`** | Required dependency. Has caused issues in recent versions: v2.6.4 had a preprocessing regression (`RuntimeError: Error while creating the NDArray`). Resolved in v2.7.0. |
| **Overwriting torch** | Installing nnunetv2 can silently replace your CUDA torch with a CPU build. Always install PyTorch first. |

For the DTU HPC cluster environment, ensure the virtual environment is activated before
any `nnUNetv2_*` commands.

## Environment Variables

Three variables are **required**. There are no others — if any is unset, nnU-Net crashes
immediately with a `RuntimeError`.

Set them permanently in `~/.bashrc`:

```bash
export nnUNet_raw="/work3/s225224/nnunet/raw"
export nnUNet_preprocessed="/work3/s225224/nnunet/preprocessed"
export nnUNet_results="/work3/s225224/nnunet/results"
```

Verify with:
```bash
echo $nnUNet_raw
echo $nnUNet_preprocessed
echo $nnUNet_results
```

| Variable | Purpose | Performance note |
|---|---|---|
| `nnUNet_raw` | Where you place raw datasets in `DatasetXXX_Name` format | — |
| `nnUNet_preprocessed` | Where preprocessed data is stored (generated by `plan_and_preprocess`) | **Should be on a fast drive (NVMe SSD recommended)** |
| `nnUNet_results` | Where trained model weights, configs, and debug output are saved | — |

> **Caveat:** These variables must be set in every shell session and in every LSF job script.
> Forgetting them causes nnU-Net to fail with a `RuntimeError` pointing to the docs.
> Add them to your job scripts explicitly (see [03 — Training](03_training.md)).

### Additional Environment Variables

| Variable | Default | Purpose |
|---|---|---|
| `nnUNet_compile` | `true` (on CUDA) | Controls `torch.compile`. Enabled by default on CUDA since v2.5+. Set to `false` to disable. First-epoch compilation adds extra startup time. |
| `nnUNet_wandb_enabled` | `0` | Set to `1` to enable built-in W&B logging (see [04 — Experiment Tracking](04_experiment_tracking.md)). |
| `nnUNet_wandb_project` | `nnunet` | W&B project name when W&B is enabled. |
| `nnUNet_wandb_mode` | `online` | W&B mode (`online` or `offline`). |

## Dataset ID Convention

nnU-Net v2 uses `DatasetXXX_Name` where `XXX` is a zero-padded 3-digit integer.

| ID | Name |
|---|---|
| `001` | `CSpineSeg` |

Full directory name: **`Dataset001_CSpineSeg`**

> **Note:** IDs 001-010 are conventionally reserved for the Medical Segmentation Decathlon
> datasets. Using `001` works but could conflict if you ever download MSD pretrained models.
> The official recommendation is to start custom datasets at ID 500+. For this project,
> `001` is fine since we are not using MSD models.

On the command line, you can refer to datasets by just the integer ID (e.g., `-d 1`), and
nnU-Net resolves the full `DatasetXXX_Name` by scanning all three env var directories.
